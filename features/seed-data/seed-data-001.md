# Task: seed-data-001

## Objective

Implement comprehensive seed data generation in `priv/repo/seeds.exs` to populate the development database with realistic test data for QA purposes. The seed script must be idempotent and run automatically during devcontainer setup.

## Context

- The app name is `Acai` with the repo module `Acai.Repo`
- The seeds file (`priv/repo/seeds.exs`) is currently a stub with only placeholder comments
- The devcontainer `postcreate.sh` already runs `mix ecto.setup`, which includes `mix run priv/repo/seeds.exs` — so no changes needed there
- The `Acai.Accounts` context provides `register_user/1` for user creation (takes `%{email: ...}`)
- User confirmation is done via `User.confirm_changeset/1` which sets `confirmed_at`
- Password is set via `User.password_changeset/2` (takes user struct and `%{password: ...}`)
- The `Acai.Teams` context provides `create_team/2` (takes `current_scope` and `%{name: ...}`)
- Roles are managed via `UserTeamRole` with titles: `"owner"`, `"developer"`, `"readonly"`
- `Acai.Accounts.Scope` provides `Scope.for_user(user)` to build scope structs needed by context functions
- All UUIDs are UUIDv7 (`Acai.UUIDv7`), auto-generated by schemas
- The test fixtures in `test/support/fixtures/data_model_fixtures.ex` demonstrate how to create all entity types at the Repo level — use similar patterns in the seed file

## Key Schema & Context Reference

| Entity | Schema Module | Context API | Key Fields |

| Team | `Acai.Teams.Team` | `Acai.Teams.create_team/2` | `name` |
| UserTeamRole | `Acai.Teams.UserTeamRole` | (direct insert) | `team_id`, `user_id`, `title` |
| Spec | `Acai.Specs.Spec` | `Acai.Specs.create_spec/3` | `team_id`, `repo_uri`, `branch_name`, `path`, `last_seen_commit`, `parsed_at`, `feature_name`, `feature_product`, `feature_description` |
| Requirement | `Acai.Specs.Requirement` | `Acai.Specs.create_requirement/2` | `spec_id`, `group_key`, `group_type` (`:COMPONENT`/`:CONSTRAINT`), `local_id`, `parent_local_id`, `definition`, `note`, `is_deprecated`, `replaced_by`, `feature_name` |
| Implementation | `Acai.Implementations.Implementation` | `Acai.Implementations.create_implementation/3` | `spec_id`, `team_id`, `name`, `description`, `is_active` |
| TrackedBranch | `Acai.Implementations.TrackedBranch` | `Acai.Implementations.create_tracked_branch/2` | `implementation_id`, `repo_uri`, `branch_name` |
